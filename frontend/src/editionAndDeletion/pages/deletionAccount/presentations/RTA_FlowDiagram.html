Flujo Operativo: ProceedButton & RTA

Interacción entre UI, Hooks y Lógica de Negocio

UI COMPONENT

ProceedButtonUI

El usuario hace clic. El botón entra en estado isExecutingDeletion (disabled) para evitar peticiones duplicadas.

HOOK

useRTAImpactAndDeletion

Captura el ImpactReport actual y construye el payload RTA: { deletionType: 'RTA', affectedAccounts, targetAccount }.

BACKEND

API /executeAccountDeletion

Inicia transacción SQL. Aplica Retrospective Annulment: Reajusta saldos históricos y elimina el registro de forma atómica.

OK

SuccessMessageUI

El hook actualiza el Store global eliminando la cuenta. Se dispara el mensaje de éxito con auto-cierre.

FAIL

ReportErrorUI

Si falla la red o validación, se captura el error y se muestra el componente de alerta con opción de reintento.
