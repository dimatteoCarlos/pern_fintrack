Análisis profundo del flujo de datos y la interacción entre componentes y hooks.

01. Estructura y Orquestación

1

AccountDeletionPage.tsx

Controlador principal que gestiona el estado del reporte y la visibilidad del modal.

2

useRTAImpactAndDeletion.ts

Hook de lógica de negocio. Orquesta las llamadas GET (reporte) y DELETE (ejecución).

3

UI Components

Componentes atómicos (ImpactReport, Modal, SuccessMessage) que renderizan el estado puro.

02. Interacción entre Funciones

¿Cómo se comunican las funciones internas del módulo?

Fase de Consulta

  • Valida accountId al inicio.
  • Ejecuta GET via useFetch.
  • Mapea la respuesta al estado local.

Fase de Ejecución

  • Construye el payload RTA.
  • Dispara POST/DELETE via useFetchLoad.
  • Notifica éxito al Store Global.

Post-Acción

  • Cierra modal automáticamente.
  • Muestra feedback visual temporal.
  • Limpia timers de seguridad.

03. Hooks de Utilidad (UX)

useEscapeKey

Permite cerrar el modal presionando 'ESC', mejorando la accesibilidad.

useAutoClose

Gestiona el cierre automático de mensajes de éxito tras 3 segundos.

useClickOutside

Detecta interacciones fuera del modal para colapsarlo.

useLangTranslation

Provee acceso reactivo al diccionario de idiomas.
