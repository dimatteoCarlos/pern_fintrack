Retrospective Total Annulment (RTA)

Un sistema de borrado lógico diseñado para revertir el impacto financiero de una cuenta en toda la cadena histórica de transacciones.

¿Qué es RTA?

Retrospectiva

Analiza hacia atrás todas las transacciones vinculadas a la cuenta objetivo.

Total

No deja residuos; ajusta los balances de las cuentas contrapartes para neutralizar el impacto.

Anulación

En lugar de solo borrar, emite una señal de ajuste que garantiza que la contabilidad cuadre.

Componentes de Soporte RTA

ReportErrorUI

Captura fallos críticos durante el cálculo del impacto RTA (ej. pérdida de conexión o ID inválido).

  • Traduce códigos de error API
  • Ofrece reintento de carga

ProceedButtonUI

El disparador del borrado. Implementa seguridad reactiva para evitar ejecuciones múltiples.

  • Estado 'disabled' en ejecución
  • Feedback visual de carga

SuccessMessageUI

Notificación post-borrado. Confirma que la base de datos ha sido purgada y ajustada.

  • Feedback positivo animado
  • Cierre automático temporizado
